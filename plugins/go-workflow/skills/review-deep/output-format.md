# Review Output Formats

Loaded by `SKILL.md` Steps 5 and 7 when the agent renders findings to the
chat or posts them to a PR. All tables and the PR-comment template live here
so the trunk stays a procedural skeleton.

## Findings Table

```
## Deep Review Findings -- N issues

| # | Priority | Category | File | Lines | Title | Confidence |
|---|----------|----------|------|-------|-------|------------|
| 1 | P0 | correctness | api/handler.go | 42-45 | Nil pointer on empty response | 0.95 |

**Overall:** patch is correct / patch is incorrect
**Explanation:** ...
**Quality Score:** 82/100
```

Priority values: `P0` (critical correctness/security), `P1` (high), `P2` (medium), `P3` (nit). Categories include: `correctness`, `security`, `performance`, `maintainability`, `dx`, `idiom`, `tests`, `docs`. Confidence is a 0.0–1.0 float (see scoring guide in `review-criteria.md`).

## Spec Compliance Section (when issue/PR context available)

```
## Spec Compliance

| Criterion | Implemented | Tested | Evidence |
|-----------|-------------|--------|----------|
| <requirement> | YES/NO/PARTIAL | YES/NO | file:line |

### Missing Requirements
- <any requirements from the issue not addressed>

### Scope Creep
- <any changes not requested> (or "None detected")
```

## Review Comments Status (when review threads available)

```
## Review Comments Status

| Thread | File:Line | Reviewer | Status |
|--------|-----------|----------|--------|
| <summary> | file.go:42 | @reviewer | Addressed / Still open |
```

## Recommendation Values

Choose exactly one for the trunk's Step 7 summary and any PR comment posted in Step 7:

- `APPROVE` — patch is correct, no actionable findings remain
- `REQUEST_CHANGES` — at least one P0/P1 finding remains unaddressed
- `COMMENT` — only P2/P3 findings, or context that the author should consider but isn't blocking

## PR Comment Template (auto-post path)

```bash
gh pr comment "$PR_NUM" --body "$(cat <<'EOF'
## Deep Review Results

<formatted findings table>
<spec compliance section if available>
<review comments status if available>
<quality score>
<recommendation>

---
*Generated by gopher-ai review-deep*
EOF
)"
```

The trailing footer (`*Generated by gopher-ai review-deep*`) is intentional —
it lets future review-deep runs identify and supersede previous review
comments.
